---
title: Store Memory
api: POST https://api.remem.online/memories
description: Embed and store a memory for a user and agent pair.
---

## Overview

Call this whenever your agent learns something worth remembering about a user. Remem embeds the text, checks for duplicates, and saves it. If a near-identical memory already exists the store is skipped automatically — no duplicate facts in your database.

---

## Request

### Headers

| Name | Required | Value |
|------|----------|-------|
| `X-API-Key` | Yes | Your Remem API key (`rm_live_xxx`) |
| `Content-Type` | Yes | `application/json` |

### Body Parameters

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `content` | string | Yes | — | The memory text to store. Min 1 character, max 10,000 characters. |
| `user_id` | string | Yes | — | Identifier for the end user this memory belongs to. |
| `agent_id` | string | Yes | — | Identifier for the agent storing the memory. |
| `memory_type` | string | No | `episodic` | One of `episodic`, `semantic`, or `summary`. |
| `importance` | float | No | `0.5` | Priority score from `0.0` (low) to `1.0` (high). Affects retrieval ranking. |
| `metadata` | object | No | `{}` | Any extra key-value data to store alongside the memory. |
| `ttl_days` | integer | No | `null` | Days until memory auto-expires. `null` means permanent. |

```bash curl
curl -X POST https://api.remem.online/memories \
  -H "X-API-Key: rm_live_xxx" \
  -H "Content-Type: application/json" \
  -d '{

    "user_id":     "user_123",
    "agent_id":    "support_bot",
    "memory_type": "semantic",
    "importance":  0.8
  }'
```

```python Python SDK
from remem import RememClient

client = RememClient(api_key="rm_live_xxx")

result = client.remember(
    "User prefers concise bullet points",
    user_id="user_123",
    agent_id="support_bot",
    memory_type="semantic",
    importance=0.8,
)

print(result.id)             # memory UUID
print(result.is_duplicate)   # False — stored successfully
```

## Response

### 201 — Memory Stored

```json
{
  "id":      "775263ee-73af-4416-9804-1f274048ae08",
  "message": "Memory stored successfully"
}
```

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | UUID of the stored memory. Save this if you need to update or delete it later. |
| `message` | string | Confirmation message. |

### 409 — Duplicate Detected

```json
{
  "detail": "Duplicate memory detected — skipped."
}
```

A near-identical memory already exists for this `user_id` and `agent_id`. Nothing was stored. This is safe to ignore in most cases — it means Remem protected your database from storing the same fact twice.

---

## Error Responses

| Code | Meaning | Fix |
|------|---------|-----|
| `401` | Invalid or missing API key | Check your key starts with `rm_` |
| `402` | Plan memory limit reached | Upgrade at remem.online/pricing |
| `409` | Duplicate memory — skipped | Safe to ignore |
| `422` | Validation error | Check your request body — `content`, `user_id`, and `agent_id` are required |

## Notes

<AccordionGroup>
  <Accordion title="How duplicate detection works">
    Before storing, Remem embeds the new content and searches for existing memories
    with cosine similarity above 0.95. If any match is found the store is skipped
    and a 409 is returned. This prevents "User is in Lagos" from being stored
    50 times across 50 sessions.
  </Accordion>
  <Accordion title="Choosing memory_type">
    Use `episodic` for things that happened — events, conversations, complaints.
    Use `semantic` for stable facts — location, preferences, account type.
    Leave `summary` for Remem to generate automatically.
    See [Memory Types](/concept/memory-types) for a full guide.
  </Accordion>
  <Accordion title="Setting importance">
    High importance (0.8–1.0) — facts you always want surfaced.
    Default (0.5) — normal memories, let recency and relevance determine ranking.
    Low importance (0.1–0.3) — low signal events that should rarely surface.
  </Accordion>
  <Accordion title="Using TTL for temporary context">
    Set `ttl_days` for session-specific context that should disappear automatically:

    ```python
    client.remember(
        "User is currently asking about order #4521",
        user_id="user_123",
        agent_id="support_bot",
        ttl_days=1,   # gone tomorrow
    )
    ```

    The memory is excluded from search results after it expires.
    It is physically deleted by a background job that runs hourly.
  </Accordion>
</AccordionGroup>
