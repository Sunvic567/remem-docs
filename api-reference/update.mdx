---
title: Update Memory
api: PATCH https://api.remem.online/memories/{memory_id}
description: Update the content of an existing memory.
---

## Overview

When a fact changes, update the existing memory instead of creating a new one. Remem re-embeds the new content so future searches reflect the update. The memory ID stays the same.

**Example:** User says "I moved from Lagos to Abuja." Find the old location memory and update it — no duplicate memories, no conflicting facts.

---

## Request

### Headers

| Name | Required | Description |
|------|----------|-------------|
| `X-API-Key` | Yes | Your Remem API key |
| `Content-Type` | Yes | `application/json` |

### Path Parameters

| Parameter | Description |
|-----------|-------------|
| `memory_id` | UUID of the memory to update. Get this from store or search responses. |

### Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `user_id` | string | Yes | Must match the user who owns this memory. |
| `agent_id` | string | Yes | Must match the agent that stored this memory. |
| `new_content` | string | Yes | Replacement text. Gets re-embedded automatically. |
| `importance` | float | No | New importance score. Keeps existing value if not provided. |
| `metadata` | object | No | New metadata. Replaces existing metadata if provided. |

<CodeGroup>
```bash curl
curl -X PATCH https://api.remem.online/memories/775263ee-73af-4416-9804-1f274048ae08 \
  -H "X-API-Key: rm_live_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "user_id":     "user_123",
    "agent_id":    "support_bot",
    "new_content": "User moved from Lagos to Abuja, Nigeria",
    "importance":  0.95
  }'
```

```python Python SDK
updated = client.update(
    memory_id="775263ee-73af-4416-9804-1f274048ae08",
    user_id="user_123",
    agent_id="support_bot",
    new_content="User moved from Lagos to Abuja, Nigeria",
    importance=0.95,
)

print(updated.id)       # same ID
print(updated.content)  # new content
```
</CodeGroup>

## Response

### 200 — Success

Returns the full updated Memory object.

```json
{
  "id":          "775263ee-73af-4416-9804-1f274048ae08",
  "content":     "User moved from Lagos to Abuja, Nigeria",
  "user_id":     "user_123",
  "agent_id":    "support_bot",
  "memory_type": "semantic",
  "importance":  0.95,
  "score":       null,
  "created_at":  "2026-06-01T10:00:00Z",
  "last_accessed": "2026-06-07T09:00:00Z",
  "expires_at":  null
}
```

### 404 — Not Found

```json
{
  "detail": "Memory '775263ee...' not found for user=user_123 agent=support_bot"
}
```

The `memory_id` doesn't exist, or it belongs to a different `user_id` or `agent_id`.

## Notes

<AccordionGroup>
  <Accordion title="Find the memory to update">
    Use `recall()` to find the right memory before updating:

```python
    # Find the location memory
    memories = client.recall(
        "where does this user live",
        user_id="user_123",
        agent_id="support_bot",
        top_k=1,
    )

    if memories:
        client.update(
            memory_id=memories[0].id,
            user_id="user_123",
            agent_id="support_bot",
            new_content="User moved from Lagos to Abuja",
        )
```
  </Accordion>
  <Accordion title="Update vs store new">
    Always update when a fact changes — never store a new memory alongside the old one. Two memories saying different things about the same fact will confuse your agent.

    Update → one clean fact ✅
    Store new → two conflicting facts ❌
  </Accordion>
</AccordionGroup>
