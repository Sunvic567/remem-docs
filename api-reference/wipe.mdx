---
title: Wipe All Memories
api: DELETE https://api.remem.online/memories
description: Delete all memories for a user and agent pair.
---

## Overview

Removes every memory stored for a `user_id` + `agent_id` combination. Use for GDPR right-to-erasure requests, account resets, or starting fresh.

<Warning>
  This is irreversible. All memories for the specified user and agent are permanently deleted.
</Warning>

---

## Request

### Headers

| Name | Required | Description |
|------|----------|-------------|
| `X-API-Key` | Yes | Your Remem API key |

### Query Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `user_id` | string | Yes | Wipe memories for this user. |
| `agent_id` | string | Yes | Wipe memories for this agent. |
| `confirm` | boolean | Yes | Must be `true`. Prevents accidental wipes. |

<CodeGroup>
```bash curl
curl -X DELETE \
  "https://api.remem.online/memories?\
user_id=user_123&agent_id=support_bot&confirm=true" \
  -H "X-API-Key: rm_live_xxx"
```

```python Python SDK
result = client.forget_all(
    user_id="user_123",
    agent_id="support_bot",
)
print(result.deleted)  # number of memories removed
```
</CodeGroup>

---

## Response

### 200 — Success

```json
{
  "deleted": 47,
  "message": "Deleted 47 memories."
}
```

| Field | Type | Description |
|-------|------|-------------|
| `deleted` | integer | Number of memories permanently removed. |
| `message` | string | Confirmation message. |

### Error Responses

| Code | Meaning |
|------|---------|
| `400` | `confirm` not set to `true` |
| `401` | Invalid or missing API key |
