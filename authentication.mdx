## API Keys

Every request to the Remem API requires your API key passed in the `X-API-Key` header.

```bash
curl https://api.remem.online/memories/search \
  ?query=test&user_id=u1&agent_id=bot \
  -H "X-API-Key: rm_live_xxx"
```

---

## Key Format

Remem API keys always start with `rm_`:

```
rm_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

<Warning>
  If your key does not start with `rm_` you will get a `401 Invalid API key format` error immediately before any database lookup is performed.
</Warning>

## Using Your Key Safely

<AccordionGroup>
  <Accordion title="Always use environment variables">
    Never hardcode your API key directly in your source code.
    Anyone who reads your code — including version control history — can see it.

    ```python
    # Wrong — key is visible in source code
    client = RememClient(api_key="rm_live_xxx")

    # Correct — key is read from environment
    import os
    client = RememClient(api_key=os.getenv("REMEM_API_KEY"))
    ```

    Store it in a `.env` file locally:
    ```bash
    REMEM_API_KEY=rm_live_xxx
    ```

    And in your deployment platform's environment variables section — never in your repo.
  </Accordion>

  <Accordion title="Add .env to .gitignore">
    Make sure your `.env` file is never committed to version control:

    ```bash
    # .gitignore
    .env
    .env.local
    .env.production
    ```

    If you accidentally commit a key — revoke it immediately and generate a new one.
  </Accordion>

  <Accordion title="Lost your key?">
    Email support@remem.online from your registered email address.
    We will revoke the old key and issue a new one.
    Old key stops working the moment it is revoked.
  </Accordion>
</AccordionGroup>

## How Authentication Works

Every request goes through this flow:

```
Your request arrives with X-API-Key header
        ↓
Remem checks the key starts with rm_
If not → 401 immediately, no DB lookup
        ↓
Remem hashes the key using SHA-256
        ↓
Looks up the hash in the api_keys table
If not found or revoked → 401
        ↓
Checks if your account is suspended
If suspended → 403
        ↓
Returns your tenant config
        ↓
Your request is processed
```

Your raw key is never stored anywhere — only the SHA-256 hash. Even if someone accessed the database they could not reverse the hash back to your key.

---

## Error Responses

| Code | Meaning | What to do |
|------|---------|------------|
| `401` | Missing, invalid, or revoked key | Check your key starts with `rm_` and matches what was emailed to you |
| `403` | Account suspended | Contact support@remem.online |
| `402` | Plan memory limit reached | Upgrade your plan at remem.online/pricing |

## Using the SDK

The SDK handles the header automatically. You only pass the key once at initialization:

```python
from remem import RememClient

client = RememClient(
    api_key="rm_live_xxx",           # passed once here
    base_url="https://api.remem.online",
)

# Every subsequent call uses it automatically
client.remember("something", user_id="u1", agent_id="bot")
client.recall("something", user_id="u1", agent_id="bot")
```

## Using the REST API Directly

If you are not using the Python SDK, pass the key as a header on every request:

```bash
# Every request needs this header
-H "X-API-Key: rm_live_xxx"
```

```javascript
// JavaScript / fetch
const response = await fetch("https://api.remem.online/memories", {
  method: "POST",
  headers: {
    "X-API-Key":    "rm_live_xxx",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    content:  "User prefers dark mode",
    user_id:  "user_123",
    agent_id: "support_bot",
  }),
});
```

```python
# Python / httpx without SDK
import httpx

resp = httpx.post(
    "https://api.remem.online/memories",
    headers={"X-API-Key": "rm_live_xxx"},
    json={
        "content":  "User prefers dark mode",
        "user_id":  "user_123",
        "agent_id": "support_bot",
    },
)
```
