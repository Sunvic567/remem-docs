---
title: How Retrieval Works
description: Understanding Remem's hybrid scoring system.
---

When you call `recall()` or `context()`, Remem doesn't just return the most similar memories — it ranks them using a hybrid score that combines three signals.

---

## The Formula
final_score = 0.70 × cosine_similarity
+ 0.20 × recency_score
+ 0.10 × importance_score

---

## The Three Signals

### Cosine Similarity (70%)

How semantically similar is the stored memory to your query?

This is what makes semantic search work. You ask "where does this user live?" and the memory "User is based in Lagos, Nigeria" scores high even though none of those words match your query — because the *meaning* matches.

### Recency Score (20%)

How recently was the memory stored or accessed?

Newer memories score higher. This prevents stale memories from outranking fresh ones. The score decays exponentially over time — a memory from yesterday scores much higher than an identical one from 6 months ago.

### Importance Score (10%)

How important did you mark this memory when storing it?

You control this with the `importance` parameter (0.0 to 1.0). High importance memories get a small boost in retrieval. Use this for facts you always want surfaced.

---

## Real Example

Say a user has three memories:

| Memory | Cosine | Recency | Importance | Final |
|--------|--------|---------|------------|-------|
| "User is in Lagos" (stored today) | 0.89 | 0.99 | 0.80 | **0.88** |
| "User is in Lagos" (stored 6 months ago) | 0.89 | 0.12 | 0.80 | **0.72** |
| "User prefers dark mode" (stored today) | 0.41 | 0.99 | 0.70 | **0.56** |

Query: "where does this user live?"

The most recent Lagos memory wins — even though the content is identical to the old one, recency pushes it to the top.

---

## Score Detail

Every search result includes a breakdown of how it scored:

```python
memories = client.recall("where does this user live?", user_id="u1", agent_id="bot")

for m in memories:
    print(m.score)         # 0.8801
    print(m.score_detail)
    # {
    #   "cosine":    0.8900,
    #   "recency":   0.9900,
    #   "importance": 0.8000,
    #   "final":     0.8801
    # }
```

Use `score_detail` to debug why a memory ranked where it did.

---

## Minimum Score Threshold

Control what comes back with `min_score`:

```python
# Strict — only highly relevant memories
memories = client.recall(query, user_id, agent_id, min_score=0.85)

# Loose — return anything remotely relevant
memories = client.recall(query, user_id, agent_id, min_score=0.50)

# Everything — no filter
memories = client.recall(query, user_id, agent_id, min_score=0.0)
```

Default is `0.70` — a good balance for most use cases.
