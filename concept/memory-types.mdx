---
title: Memory Types
description: Understanding episodic, semantic, and summary memories.
---

Remem supports three memory types. Choosing the right one helps your agent retrieve the most useful context.

---

## Episodic

Things that **happened** — events, conversations, actions.

```python
client.remember(
    "User complained that delivery took 3 weeks",
    user_id="user_123",
    agent_id="support_bot",
    memory_type="episodic",
)
```

<Info>
  Use episodic for conversation events, support tickets, user actions — anything time-specific.
</Info>

---

## Semantic

**Facts about the user** — preferences, profile info, stable knowledge.

```python
client.remember(
    "User is based in Lagos, Nigeria",
    user_id="user_123",
    agent_id="support_bot",
    memory_type="semantic",
    importance=0.9,   # high importance — stable fact
)
```

<Info>
  Use semantic for user profile facts, preferences, and anything that doesn't expire quickly.
  Set higher importance scores for facts you always want retrieved.
</Info>

---

## Summary

**Compressed older memories** — auto-generated by Remem when episodic count gets large.

You don't need to create these manually. Remem generates them automatically to keep retrieval fast and costs low.

---

## Choosing the Right Type

| Situation | Type | Example |
|-----------|------|---------| 
| Something the user said | `episodic` | "User said they hate emails" |
| A fact about the user | `semantic` | "User is a developer" |
| User preference | `semantic` | "User wants responses in bullet points" |
| Something that happened | `episodic` | "User placed order #4521" |
| Compressed old context | `summary` | Auto-generated |

---

## Filtering by Type

```python
# Only retrieve semantic memories
memories = client.recall(
    "user preferences",
    user_id="user_123",
    agent_id="support_bot",
    memory_type="semantic",
)
```
